Why?

Readers benefit from interactivity because they have different amounts of background and are interested in different parts of an article. Letting users drill down into topics that interest them or get background when they need it allows an author to serve a diverse audience in a more personal manner. For example (...)this post attempts to address two topics, why articles should be move interactive and how content creators can use my script to bring their sites to life. These are very different topics with very different audiences, but because readers have some control over what content is shown I can serve all comers. (another reason?)

The other big case for interactive articles is it lets authors collect data on their readers. (another reason?)

With Pn you can make two kinds of elements, clickable elements and showable elemets.

A clickable element is defined by the attribute data-pn-id, data-pn-id is used to uniquely identify a clickable item, In the above example the clickable item is identified by "example". A clickable item keeps track of if it is "on" or "off" (true or false) . A items starts "off" and toggles when it is clicked. (data-pn-id naming rules)data-pn-id's can not include the character '(',')','!' or ' ' and can not be "and" or "or".

A showable element is identified by the attribute data-pn-if, data-pn-if contains a logical expression, When this expression evaluates to true the contents of the tag are shown. The expression consisting of logical operations and the identifiers of clickable elements. (examples) For example data-pn-if could be "some-identifier" or "some-indentifer and another-indentifer" or "some-indentifer and (another-andentifer or third-identifier)" The identifier for a clickable element is evaluated to true when its clickable element is on and evaluated to false otherwise. In the example above, the showable element has the value example for it's data-pn-if this mean it will be shown when the clickable item with a data-pn-id of "example" (the span right above it) is on.

CSS

I have included several CSS classes to allow users to style there pn elements.
  • pn-id: applied to all elements with the attribute data-pn-id
  • pn-on: applied to all elements with the attribute data-pn-id that are currently on.
  • pn-off: applied to all elements with the attribute data-pn-id that are currently off.
  • pn-if: applied to all elements with the attribute data-pn-if.
  • pn-shown: applied to all elements with the attribute data-pn-if that are currently being shown.
  • pn-hidden: applied to all elements with the attribute data-pn-if that are currently being hidden.

data-pn-start-on

you can specify you want a clickable element to start "on" by adding the attribute data-pn-start-on="true"
